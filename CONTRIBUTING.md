# Contributing to Cullr

Hell yeah, PRs are more than welcome — bug fixes, features, UX polish, whatever you've got. This is a community thing and every contribution counts.

## How to Contribute

1. **Fork** the repo
2. **Create a branch** for your change (`git checkout -b fix/thing` or `feat/thing`)
3. **Make your changes** and commit with clear messages
4. **Open a Pull Request** against `main`

Before submitting, make sure:

- `npm run lint` passes with no errors
- Your code follows the existing patterns and conventions in the codebase
- New features include a brief description in your PR of what they do and why

## Reporting Issues

Found a bug or have an idea? Open an issue with:

- What you expected vs. what happened
- Steps to reproduce (if applicable)
- Device info (Android version, phone model)

## Code of Conduct

Be respectful. No harassment, discrimination, or toxicity. Keep it constructive.
